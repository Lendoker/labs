# Лабораторні роботи — Алгоритми сортування

Веб-проєкт для виконання лабораторних робіт з алгоритмів сортування: інтерактивна візуалізація, покроковий режим, динамічна аналітика, звукові ефекти.

---

## Загальні можливості (усі лабораторні)

**Інтерактивний блок:** поле введення масиву (числа через кому або пробіл); кнопка генерації випадкового масиву (довжина та діапазон налаштовуються); кнопки «Сортувати», «Крок», «Авто-режим», «Скинути»; повзунок швидкості (мс) для авто-режиму.

**Візуалізація:** масив у вигляді вертикальних стовпчиків (висота пропорційна значенню). Кольори елементів залежать від алгоритму (відсортована частина, поточні порівнювані/переставляємі елементи, завершений стан) — див. опис кожної ЛР нижче.

**Аналітика:** порівняння та перестановки оновлюються на кожному кроці; час виконання (мс) і позначення складності алгоритму.

**Додатково:** звуковий «тик» на кроці та мелодія при завершенні; фіксована бокова панель; світла/темна тема.

---

## Лабораторна робота №1 — Сортування вибором

**Алгоритм:** На кожній ітерації серед невідсортованих елементів знаходимо мінімальний і ставимо його на поточну позицію. Відсортована частина зростає зліва направо.

**Теорія:** принцип роботи, псевдокод, складність O(n²) у всіх випадках.

**Кольори візуалізації:** зелений — відсортована частина; жовтий — поточний мінімальний (кандидат на перестановку); синій — елемент порівняння; блакитний — після завершення.

---

## Лабораторна робота №2 — Сортування включенням

**Алгоритм:** Масив розділяється на відсортовану (зліва) та невідсортовану (справа) частини. На кожному кроці один елемент з невідсортованої частини вставляється на потрібну позицію у відсортовану шляхом порівнянь і зсувів.

**Теорія:** поділ на дві частини (риска |), приклад сортування, псевдокод; складність O(n²) у середньому/найгіршому, O(n) у найкращому.

**Завдання за рівнями:** середній — відсортувати вручну та програмно; достатній — інтерактив і візуалізація; високий — інша мова або графічний інтерфейс.

**Кольори візуалізації:** зелений — відсортована частина; жовтий — елемент, який вставляється (key); синій — елемент порівняння з key; блакитний — після завершення.

---

## Лабораторна робота №3 — Сортування бульбашкою

**Алгоритм:** Порівнюються попарно сусідні елементи; якщо порядок неправильний — міняються місцями. На кожній ітерації зовнішнього циклу найбільший елемент «спливає» до кінця масиву.

**Теорія:** принцип роботи, псевдокод; складність O(n²) у середньому/найгіршому, O(n) у найкращому (вже відсортований масив).

**Кольори візуалізації:** зелений — відсортована частина (справа); жовтий/синій — пара елементів, які порівнюються або міняються; блакитний — після завершення.

---

## Лабораторна робота №4 — Сортування злиттям

**Алгоритм:** Підхід «розділяй і володарюй». Масив розділяється на частини, які рекурсивно (або в цій реалізації — ітеративно, bottom-up) сортуються і потім зливаються в один відсортований масив.

**Теорія:** ітеративний підхід (крок h подвоюється: злиття підмасивів розміром 1, 2, 4, …); складність O(n log n) у всіх випадках.

**Кольори візуалізації:** відображаються поточні підмасиви та індекси злиття; блакитний — після завершення.

## Структура проєкту

```
src/
├── core/algorithms/     # Чиста алгоритмічна логіка (без DOM)
│   ├── selectionSort.js
│   ├── insertionSort.js
│   ├── bubbleSort.js
│   └── mergeSort.js
├── utils/               # Утиліти (парсинг, звук, тема)
├── ui/
│   ├── components/      # Сторінки лабораторних (Lab1Page … Lab4Page)
│   ├── layout/          # App, Sidebar
│   └── styles/          # CSS, Tailwind
├── main.js              # Точка входу
└── index.html
```

## Встановлення та запуск

```bash
npm install
npm run dev
```

Сайт буде доступний на `http://localhost:10000`.

**Збірка для продакшену:**

```bash
npm run build
```

Готові файли — у папці `dist/`.

**Деплой на Render.com:** Build — `npm install && npm run build`; Start — `npm start` (сервер слухає `PORT`, прив’язаний до `0.0.0.0`). У корені проєкту є `render.yaml`.
