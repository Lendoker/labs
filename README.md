# Лабораторні роботи — Алгоритми сортування

Веб-проєкт для виконання лабораторних робіт з алгоритмів сортування. Інтерактивна візуалізація, покроковий режим, динамічна аналітика, звукові ефекти.

---

## Лабораторна робота №1 — Сортування вибором

### Що відбувається

**Алгоритм:** На кожній ітерації серед невідсортованих елементів знаходимо мінімальний і ставимо його на поточну позицію. Відсортована частина поступово зростає зліва направо.

**Теоретичний блок:**
- Пояснення принципу роботи алгоритму
- Опис складності O(n²) (середній, найгірший, найкращий випадки однакові)
- Псевдокод у форматі циклів

**Інтерактивний блок:**
- Поле введення масиву (числа через кому або пробіл)
- Кнопка генерації випадкового масиву (12 елементів, 1–100)
- Кнопка «Сортувати» — запуск алгоритму, оновлення статистики
- Кнопка «Крок» — покрокове виконання (доступна після сортування)
- Кнопка «Авто-режим» — автоматичне відтворення з регульованою швидкістю
- Кнопка «Скинути» — очищення візуалізації та статистики
- Повзунок швидкості (50–1000 мс) — затримка між кроками в авто-режимі

**Візуалізація:**
- Масив у вигляді вертикальних стовпчиків (висота пропорційна значенню)
- **Зелений** — відсортована частина
- **Жовтий** — поточний мінімальний елемент (кандидат на перестановку)
- **Синій** — елемент, з яким відбувається порівняння
- **Блакитний** — усі стовпчики після завершення сортування

**Аналітика:**
- Порівняння — оновлюється на кожному кроці
- Перестановки — змінюється при обміні елементів
- Час (мс) — час виконання алгоритму
- Складність — O(n²)

**Додатково:** Звуковий «тик» на кожному кроці, мелодія при завершенні. Бокова панель завжди видима (fixed). Світла/темна тема.

## Лабораторна робота №2 — Сортування включенням

### Що відбувається

**Алгоритм:** Масив розділяється на відсортовану (зліва) та невідсортовану (справа) частини. На кожному кроці береться один елемент з невідсортованої частини і вставляється на потрібну позицію у відсортовану шляхом порівнянь і зсувів.

**Мета роботи:** Навчитися сортувати масиви методом включенням (insertion sort).

**Завдання за рівнями:**
- **Середній:** Відсортувати вручну, реалізувати програмно
- **Достатній:** Додати інтерактив (введення, випадковий масив, покрокова візуалізація, підрахунок операцій, час)
- **Високий:** Реалізувати іншою мовою або додати графічний інтерфейс

**Теоретичний блок:**
- Методичні рекомендації (поділ на дві частини, риска | між ними)
- Приклад сортування: `6 | 5 3 1 8 7 2 4` → … → `1 2 3 4 5 6 7 8`
- Псевдокод алгоритму
- Складність O(n²) у середньому/найгіршому, O(n) у найкращому

**Інтерактивний блок:**
- Той самий набір елементів, що й у ЛР №1 (введення, випадковий масив, сортувати, крок, авто-режим, скинути, повзунок швидкості)

**Візуалізація:**
- **Зелений** — відсортована частина (елементи ліворуч від «позиції вставки»)
- **Жовтий** — елемент, який зараз вставляється (key)
- **Синій** — елемент, з яким порівнюється key
- **Блакитний** — усі стовпчики після завершення

**Аналітика:**
- Порівняння — динамічно на кожному кроці
- Перестановки (зсуви) — при перенесенні елементів
- Час (мс), складність O(n²)

**Додатково:** Звукові ефекти, фіксована бокова панель, теми.

## Структура проєкту

```
src/
├── core/algorithms/     # Чиста алгоритмічна логіка (без DOM)
│   ├── selectionSort.js
│   └── insertionSort.js
├── utils/               # Утиліти (парсинг, звук, тема)
├── ui/
│   ├── components/      # Сторінки лабораторних (Lab1Page, Lab2Page)
│   ├── layout/          # App, Sidebar
│   └── styles/          # CSS, Tailwind
├── main.js              # Точка входу
└── index.html
```
